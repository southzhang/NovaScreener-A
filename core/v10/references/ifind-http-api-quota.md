## ⚠️ 致命陷阱：high_frequency每根K线都算quota

**2026-05-13教训**：测试时对1554只股票逐只获取分钟K线，每只返回240根K线，结果：
```
1554只 × 240根 = 372,960条/扫描 × 3次/天 = 330万条/月
配额150万，一次测试就用了110万（73%）！
```

**原因**：`high_frequency` 端点按**每根K线**计费，不是按API调用次数。一只股票全天240根分钟K线 = 240条quota。

**正确用法**：
- ✅ 仅对V10信号股（~10只）获取分钟K线 → 10×240=2400条/扫描 → 15.8万/月（10.5%）
- ❌ 对所有预过滤股票（1554只）获取 → 37万条/扫描 → 超配额

**对比其他端点的计费方式**：
| 端点 | 计费单位 | 100只股票消耗 |
|------|----------|--------------|
| real_time_quotation | 100条（每只1条） | 100条 |
| high_frequency | 24,000条（每只240条） | 24,000条 ⚠️ |
| cmd_history_quotation | 100条（每只1条） | 100条 |
| snap_shot | 47,800条（每只全天tick） | 47,800条 ⚠️ |

**铁律**：分钟K线和日内快照消耗极大，仅用于TOP候选股的最终确认，绝不用于预筛选阶段。

## 最佳实践

1. **问财不限量** → 预筛选/选股/板块/验证全用问财
2. **全市场行情走腾讯** → 免费无限制，iFinD quota不够
3. **iFinD用于高价值查询** → 分钟K线/PB补充/基本面
4. **批量大小500只** → 减少API调用次数
5. **Token刷新加锁** → 防止多线程重复刷新

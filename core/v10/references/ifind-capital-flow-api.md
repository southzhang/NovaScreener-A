# iFinD MCP 资金面数据API参考

## 可用工具

### 1. `mcp_ifind_stock_get_stock_performance`

**功能**：股票行情表现+资金面数据（主力净流入/融资融券/大宗交易/龙虎榜）

**查询方式**：自然语言query，如"英唐智控300131最近3天的主力净流入额和龙虎榜上榜次数"

**返回字段**：
| 字段 | 说明 | 示例 |
|------|------|------|
| 主力净流入额（单位：元） | 当日主力资金净流入 | +2.64亿 / -2.98亿 |
| 区间龙虎榜上榜次数 | 指定日期区间内上榜次数 | 0.0 / 2.0 |
| 区间龙虎榜买入额（单位：元） | 机构/游资买入总额 | — |
| 区间龙虎榜卖出额（单位：元） | 机构/游资卖出总额 | — |
| 融资融券余额（单位：元） | 两融余额 | 20.26亿 |
| 区间大宗交易成交总额（单位：元） | 大宗交易金额 | — |
| 区间大宗交易成交总量（单位：股） | 大宗交易股数 | — |
| 净买入额占比 | 净买入占成交额比例 | — |
| 净卖出额占比 | 净卖出占成交额比例 | — |

**批量查询**：可在query中列出多只股票，用顿号分隔。

**限制**：
- 龙虎榜/大宗交易：仅当指定区间内有记录时返回数据，否则为空
- 融资融券：仅融资融券标的有数据（多为大盘股）
- 主力净流入：所有A股可查，最实用

### 2. `mcp_ifind_stock_get_stock_shareholders`

**功能**：股东信息（机构持股/北向资金）

**返回字段**：
| 字段 | 说明 | 数据频率 |
|------|------|----------|
| 沪深港通持股数量（单位：股） | 北向资金持股数 | 日度 |
| 沪深港通持股占比（单位：%） | 北向资金持股比例 | 日度 |
| 沪深港通持股占流通A股比例（单位：%） | 北向占流通盘比 | 日度 |
| 持股机构数（单位：家） | 机构持股家数 | 季度(MRQ) |
| 机构持股数量合计（单位：股） | 机构总持股 | 季度(MRQ) |
| 机构持股比例合计（单位：%） | 机构持股比例 | 季度(MRQ) |

**限制**：
- 北向资金：仅沪深港通标的（大盘蓝筹）有数据
- 机构持股：仅机构覆盖的股票有数据（中小盘可能为空）
- 数据频率：北向资金日度，机构持股季度（MRQ=最近报告期）

## ⚠️ 日期范围不一致陷阱（2026-05-14实测发现）

**问题**：同一工具 `get_stock_performance` 的不同指标，返回的日期范围可能差异巨大。在同一次query中列出多个指标时，各指标的参数说明（params）可能完全不同。

**实测案例**：

| 指标 | 你期望的范围 | 实际返回的params | 后果 |
|------|------------|-----------------|------|
| 区间龙虎榜上榜次数 | 近5天 | `截止日前1周` → 最新 | ✅ 正确 |
| 区间龙虎榜买入额/卖出额 | 近5天 | `截止日前一交易日` → 最新 | ⚠️ 仅1天 |
| 区间龙虎榜卖出额（单独query） | 近5天 | `截止日52周前` → 最新 | ❌ 返回一年数据！ |
| 区间大宗交易成交总额 | 近5天 | `截止日前一交易日` → 最新 | ⚠️ 仅1天 |

**最危险的场景**：query"近5天龙虎榜卖出额"返回的卖出额是52周累计值，而"近5天龙虎榜上榜次数"是1周值。两者放在一起比较会导致：
- 海兰信卖出额31.77亿 vs 上榜次数0 → 看似矛盾（实际是52周累计 vs 1周）
- 北投科技卖出额5.59亿 vs 上榜次数2 → 看似合理（但实际卖出额覆盖一年）

**铁律**：
1. **永远不要在同一次query中混合不同日期范围的指标** — 分开查
2. **检查返回结果的params说明** — 确认实际日期范围，不要假设
3. **龙虎榜卖出额/买入额单独查时，默认可能是52周范围** — 需要验证
4. **正确做法**：分别查询每个维度，每个维度内保持日期范围一致

**正确的三维度查询模式（2026-05-14优化版）**：
```
Query 1: "XX股票的主力净流入额和主力净流入额占比"  → 当日数据（⚠️ 不加"最新"，不加"今天"，批量≤6只）
Query 2: "XX股票的最新融资融券余额"  → 当日数据（融资融券查询"最新"无影响）
Query 3: "XX股票的近5天大宗交易成交总额和大宗交易上榜次数"  → ⚠️ 实际返回52周累计（非5天）
Query 4: "XX股票的近5天龙虎榜上榜次数"  → 确认上榜次数（5天范围较可靠）
Query 5: "XX股票的近5天龙虎榜买入额"  → ⚠️ 需验证日期范围
Query 6: "XX股票的近5天龙虎榜卖出额"  → ⚠️ 需验证日期范围
```

**批量查询的两个陷阱（2026-05-14多次实测确认）**：

**陷阱1 — 批量返回空值**：query列出股票后，API可能返回股票列表但**所有数值字段为空**（`\t`）。这不是"无数据"，而是批量查询的返回质量问题。

**陷阱2 — 批量遗漏股票**：11只股票的批量query可能只返回5-6只，其余被忽略。

**2026-05-14实测对比（同一批11只股票）：**

| 查询方式 | 结果 |
|----------|------|
| 11只批量 + "最新"关键词 | 返回5只（空值）+ 6只遗漏 ❌ |
| 6只批量（去掉"最新"） | 返回6只（**有实际数值**）✅ |
| 单只 + "今天"关键词 | 部分有值、部分空值 ⚠️ |
| 单只（无时间修饰词） | 多数有值 ✅ |

**铁律 — 批量查询最佳实践：**
1. **控制批量大小**：每批≤6只股票，不要一次查11只
2. **去掉"最新""今天"等时间修饰词**：直接写"XX股票的主力净流入额和主力净流入额占比"
3. **验证返回值**：检查是否为空值（`\t`），空值≠无数据，需要换phrasing重查
4. **分批策略**：11只→分2批（6+5），每批独立查询

**推荐查询模板（实测有效）：**
```
# ✅ 有效：6只+无时间词
"北投科技600936、金道科技301279、安彩高科600207、凯龙高科300912、南风股份300004、海兰信300065的主力净流入额和主力净流入额占比"

# ❌ 无效：11只+带"最新"
"北投科技600936、航天智装300455、...保变电气600550的最新主力净流入额和主力净流入额占比"
```

**Fallback流程**：
```python
# Step 1: 分批批量查询（每批≤6只）
batch1 = mcp(query="6只股票的主力净流入额和主力净流入额占比")
batch2 = mcp(query="另外5只股票的主力净流入额和主力净流入额占比")

# Step 2: 检查遗漏和空值，对缺失股票单独重查
expected = {'600936','301279','600207','300912','300065','300199','603617'}
returned = set(batch1.keys()) | set(batch2.keys())
missing = expected - returned  # 2026-05-15: 7只batch返回6只，300065海兰信被静默丢弃
for code in missing:
    single = mcp(query=f"{name}{code}的主力净流入额")  # 单只查询补回

for code, data in merge(batch1, batch2).items():
    if data['main_net_inflow'] is None:
        single = mcp(query=f"{name}{code}主力净流入额")
        # 单独查询通常能拿到数据
```

**2026-05-15实测 — 7只股票批量查询海兰信被丢弃**：
- Query: "600936北投科技、301279金道科技、600207安彩高科、300912凯龙高科、300065海兰信、300199翰宇药业、603617君禾股份的最新主力净流入额和主力净流入额占比"
- 返回: 6只（603617/600936/301279/600207/300912/300199），**300065海兰信完全缺失**（不在返回结果中，非空值）
- 补查: 单只query "300065海兰信的最新主力净流入额和主力净流入额占比" → 成功返回 -247,062,503元/-1.6127%
- **结论**: 7只已超过安全阈值（≤6只），即使只超1只也会静默丢弃。严格遵守≤6只规则。丢弃是静默的（不报错、不返回空值行），必须通过expected集合比对才能发现。

## ⚠️ 盘前（开盘前）数据可用性（2026-05-15实测确认）

**场景**：资金面缓存刷新cron在09:00运行，此时A股尚未开盘（9:30开盘），当日数据不可用。

**实测结果（2026-05-15 09:00，11只持仓股）**：

| 指标 | 今日(0515)数据 | 昨日(0514)数据 | 说明 |
|------|---------------|---------------|------|
| 主力净流入额 | 全部`\t`空值 | 10/11只有效 | 盘前无当日主力数据 |
| 融资融券余额 | 大部分`\t`空值 | 7/11只有效 | 部分标的盘前可查昨日 |
| 龙虎榜上榜次数 | 0 | 0 | 近5天区间不受影响 |
| 大宗交易 | 0 | 0 | 近5天区间不受影响 |

**"最新" vs 显式日期的行为差异**：

| 查询方式 | 盘前行为 | 正确？ |
|----------|---------|--------|
| `"XX股票的主力净流入额"`（无时间词） | 返回最近有数据的交易日 | ✅ 自动fallback |
| `"XX股票的最新主力净流入额"` | 同上，但批量时更易空值 | ⚠️ 不稳定 |
| `"XX股票在20260515的主力净流入额"` | 返回空值（当日无数据） | ❌ |
| `"XX股票在20260514的主力净流入额"` | 返回昨日数据 | ✅ |

**盘前采集铁律**：
1. **不加显式日期**：让API自动返回最近可用数据
2. **不加"最新""今天"等时间词**：已有铁律，盘前更关键
3. **缓存标注实际数据日期**：在JSON中加`note`字段说明数据是哪天的
4. **批量查询盘前更不可靠**：盘前批量查询空值率更高，优先单只查询
5. **Fallback流程**：批量空值 → 单只无时间词 → 单只指定昨日日期

**缓存JSON建议结构**：
```json
{
  "last_update": "2026-05-15 09:00:00",
  "data_date": "2026-05-14",
  "note": "主力净流入为5月14日数据（5月15日盘中数据尚未出），融资融券为最新，龙虎榜和大宗为近5日区间",
  "dimensions": { ... }
}
```
- `data_date`：主力净流入的实际数据日期（盘中/盘前fallback时标注，方便下游判断数据新鲜度）
- `note`：人类可读的数据说明（哪些维度是哪天的）

**读取兼容**

**现象**：部分股票的某些指标**始终返回空值**，不是查询问题，而是该股票不在该数据范围内。

**实测案例（11只持仓股，2026-05-15）**：

| 股票 | 融资融券余额 | 主力净流入金额 | 原因 |
|------|-------------|---------------|------|
| 300455 航天智装 | 始终空 | ✅ 可查 | 非融资融券标的 |
| 603617 君禾股份 | 始终空 | ✅ 可查 | 非融资融券标的 |
| 603080 新疆火炬 | 始终空 | ✅ 可查 | 非融资融券标的 |
| 600207 安彩高科 | ✅ 3.184亿 | 始终空(仅占比) | 融资融券标的，但主力金额查不到 |
| 300912 凯龙高科 | ✅ 1.6398亿 | ✅ 可查(无占比) | 融资融券标的 |

## ⚠️ 龙虎榜买入额必须单独查询（2026-05-14实测确认）

**现象**：当query中同时包含"龙虎榜上榜次数"和"龙虎榜买入额和卖出额"时，API只返回上榜次数和卖出额，**买入额被静默丢弃**。

**实测对比（11只持仓股）**：

| 查询方式 | 返回结果 |
|----------|----------|
| `"XX股票近5天龙虎榜上榜次数和龙虎榜买入额和龙虎榜卖出额"` | 只返回上榜次数+卖出额，**无买入额** ❌ |
| `"XX股票区间龙虎榜买入额"`（单独query） | 返回买入额 ✅ |
| `"XX股票区间龙虎榜卖出额"`（单独query） | 返回卖出额 ✅ |

**铁律**：龙虎榜三件套（上榜次数/买入额/卖出额）需要**至少2次query**：
```
Query A: "XX股票近5天龙虎榜上榜次数和龙虎榜卖出额"  → 上榜次数+卖出额
Query B: "XX股票区间龙虎榜买入额"                    → 买入额
```

**原因推测**：iFinD API对"买入额"字段在批量/多指标query中优先级较低，可能被截断。

## ⚠️ 主力净流入金额单位混合格式（2026-05-14实测确认）

**现象**：同一次批量查询中，不同股票的主力净流入额返回格式不一致：
- 部分返回**万元格式**：`681.2107万`、`2832.2373万`、`615.1494万`
- 部分返回**原始元**：`-47590917`、`-25633791.2`、`-247062503`

**推测规则**：金额绝对值较小时用"万"单位（<1亿），较大时用原始元（>1亿）。

**解析代码**：
```python
def parse_amount(raw_str):
    """解析主力净流入额（支持万/亿单位）"""
    if not raw_str or raw_str.strip() == '\\t':
        return None
    s = raw_str.strip()
    if s.endswith('亿'):
        return float(s[:-1]) * 100000000
    elif s.endswith('万'):
        return float(s[:-1]) * 10000
    else:
        return float(s)
```

## ⚠️ 大宗交易日期范围敏感性（2026-05-14二次验证）

同一股票在不同query措辞下返回不同的大宗交易记录数：

| 股票 | Query A "近5天大宗交易上榜次数" | Query B "区间大宗交易上榜次数" |
|------|------|------|
| 南风股份 300004 | 0 | **6** |

**原因**：Query A的"近5天"被API解释为极短窗口，而Query B的"区间"默认回溯更长时间（可能3-6个月）。

**铁律**：需要近5天大宗交易数据时，不能依赖API的日期参数（不可控）。应在代码中按实际日期过滤返回结果。datacenter-web API同理（全量历史返回）。

## ⚠️ 融资融券+大宗交易混合批量查询不可靠（2026-05-15实测确认）

**现象**：将"融资融券余额"和"大宗交易成交总额和上榜次数"放在同一次批量query中，返回结果严重不全。

**实测案例（7只股票）**：
- Query: "600936北投科技、301279金道科技、...、603617君禾股份的最新融资融券余额和近5天大宗交易成交总额和大宗交易上榜次数"
- 返回: **仅301279金道科技1只**（融资融券1.8493亿+大宗交易3笔3355.72万），其余6只完全缺失
- 而且大宗交易参数实际是52周范围（非5天）

**正确做法 — 分开查询**：
```
Query 1: 每只股票单独查"最新融资融券余额" → 逐只返回
Query 2: 每只股票单独查"近5天大宗交易成交总额和大宗交易上榜次数" → 逐只返回
```

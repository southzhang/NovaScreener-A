# 交易系统 Cron 任务依赖调度

## 核心原则

**有依赖关系的任务，基础任务必须先跑，下游任务后跑。**

## 任务依赖图

```
竞价选股(09:25) ──→ /tmp/auction_v2_stocks.txt ──→ V10尾盘扫描(14:30)优先检查
                                                         
V10全扫描(25,55分) ──→ v10_scan_results ──→ V10盘中盯盘(30,0分)读取结果
                                                         
资金面缓存(*/30分) ──→ capital_flow.json ──→ 实盘盯盘/选股读取
```

## ⚠️ 调度对齐 Pitfall

**问题**：全扫描 `25,55` 和盯盘 `0,10,20,30,40,50` 在 55→0 交接时顺序正确（55全扫描→0盯盘），但在 25→30 也正确（25全扫描→30盯盘）。**但 50 分的盯盘会在 55 分全扫描之前跑**，用的是旧数据。

**修复方案**：盯盘改成 `30,0`，确保每次都在全扫描之后：
- 09:25 全扫描 → 09:30 盯盘 ✓
- 09:55 全扫描 → 10:00 盯盘 ✓

**铁律**：修改任何交易任务的 schedule 时，必须检查依赖任务的时间是否仍然对齐。

# 腾讯行情 qt.gtimg.cn 字段映射表

## 调用方式
```bash
curl -s "https://qt.gtimg.cn/q=sh600936,sz300912,sz300065,sz002156,sh603986,sz002371,sz002409" | iconv -f gbk -t utf-8
```

每只股票返回一行，格式: `v_{code}=\"...~...~...\"`，以 `~` 分隔字段。

## ⚠️ 扩展字段（索引49-59）—— 主力净流入字段可靠性

### 索引49：量比
- **定义**：当日均量/近5日均量比率
- **可靠性**：✅ 高，调用了腾讯的实时量比计算

### 索引50：主力净流入（万元）—— ❌ 不可靠，实测存疑

**最初认为是** 主力净流入（大单净额），单位万元。正=净流入，负=净流出。

**⚠️ 2026-05-27实测更新**：field[50] 的值在多个场景下**极小且不合理**，与股票的实际成交金额和资金流向存在数量级偏差。例如正业科技(300410)当日成交额8亿+，field[50]仅显示1.99万元——不可能是一个"主力净流入"级别的数值。

**根因推测**：
1. **非资金流向字段**：field[50] 可能根本不是主力净流入，而是另一个含义未明的指标（部分股票的field[50]在总市值量级，部分在几万元量级）
2. **Level-1估算精度极低**：即使是主力净流入，腾讯基于Level-1 tick数据的估算精度也远低于同花顺Level-2逐笔统计
3. **索引偏移**：不同股票/不同时段，parts[50]可能对应不同的指标（偶尔是总市值/流通市值量级，偶尔是万元量级）

**不要用 field[50] 做任何资金面决策判断！**
- 方向（正/负）也**不可信**（跟真正的资金流向方向可能不一致）
- 不作为"主力净流入>0是硬指标"的依据
- 不要在选股六关验证中引用 field[50] 的资金面数据
- `refresh_capital_flow.py` 中如果还在用 parts[49] 获取 field[50]，应当换成其他更可靠的数据源

### 索引51：流通市值 or 主力净流入——含义不确定

field[51] 的值范围更广：在正业科技(300410)的案例中显示 -2465，这是一个更接近真实主力净流入（万元）的数量级；而在其他股票上可能是流通市值（千万/亿元级）。

**建议**：如果非要用腾讯API估算主力净流入方向，**先用索引51**（数量级更匹配），但同样只能看方向（正/负），不能比数值大小。

### ✅ 正确做法：用腾讯 `ff_` 资金流向专用接口

腾讯实际上有专门的主力资金流向接口：

```
# 深圳市场
https://ifzq.gtimg.cn/appstock/app/fqkline/getff?_var=ff_kline&param=sz300410,day,,,30

# 上海市场
https://ifzq.gtimg.cn/appstock/app/fqkline/getff?_var=ff_kline&param=sh600000,day,,,10
```

⚠️ **局限性**：
- 仅在交易时段返回有效数据（开盘~收盘），盘后返回空
- 深市和沪市URL格式相同（前缀不同）
- 返回格式：`ff_data` 嵌套结构，含 `tickTime` 时间戳和每tick的净流入
- 未全面实测，作为后续数据源补充方案备查

## 关键注意事项

1. **MA字段不可信**: parts[42]/43/44 的MA值对部分股票严重失真（如信凯健康MA10显示10.64而实际75.35），必须从K线API自己计算
2. **成交额单位**: parts[36] 单位是元（全数字），parts[37] 是量价额组合 `现价/成交量(手)/成交额(元)`，用 `.split('/')[2]` 取成交额再除以1e8得亿元
3. **涨跌幅**: parts[32] 直接用，已验证准确
4. **编码**: 返回是 GBK，需 `iconv -f gbk -t utf-8`
5. **PE(TTM)部分索引**: 实测 parts[39] 返回PE值。亏损股（净利润为负）返回负数（如-31.18），非亏损股返回正数。可用于快速判断盈利状态。
6. **🔥 主力净流入 field[50] 不靠谱**: 实测值在1.99万（正业科技成交8亿+）这种不可能的数量级上。**不要用于资金面验证。** 替代方案：腾讯 `ff_` 接口（交易时段有效）或同花顺实时API。

## 批量解析 Python 示例

```python
import subprocess
import re

codes = "sh600936,sz300912,sz300065,sz002156,sh603986,sz002371,sz002409"
url = f"https://qt.gtimg.cn/q={codes}"
result = subprocess.run(
    ["curl", "-s", url], capture_output=True, timeout=15
)
text = result.stdout.decode("gbk")  # qt.gtimg.cn 返回 GBK
...
stocks = {}
for line in text.strip().split(";"):
    if "=" not in line:
        continue
    code_part, data = line.split("=", 1)
    code = code_part.split("_")[-1]
    parts = data.strip('"').split("~")
    if len(parts) < 35:
        continue
    stocks[code] = {
        "name": parts[1],
        "price": float(parts[3]),
        "prev_close": float(parts[4]),
        "change_pct": float(parts[32]),
        "high": float(parts[33]),
        "low": float(parts[34]),
        "volume": int(parts[6]),
        "amount": int(parts[36]),  # 元
        "pe": float(parts[39]) if parts[39] else 0,  # PE(TTM), 负数=亏损
    }
```

## 买卖压力分析（外盘/内盘）

利用 parts[7](外盘/主动买入) 和 parts[8](内盘/主动卖出) 判断当日主力资金方向：

```python
outer = int(parts[7])  # 外盘 = 主动买入（以卖价成交）
inner = int(parts[8])  # 内盘 = 主动卖出（以买价成交）
net = outer - inner     # 正=买方主导，负=卖方主导

if net > 0:
    pressure = f"买方主导(净买入{net}手)"
elif net < 0:
    pressure = f"卖方主导(净卖出{abs(net)}手)"
else:
    pressure = "买卖均衡"
```

**实战案例（2026-05-25 10:35）**：
| 股票 | 外盘(手) | 内盘(手) | 净方向 | 股价表现 |
|------|----------|----------|--------|----------|
| 长电科技 | 810749 | 760655 | 净买入+50094 ✅ | +4.86%强势 |
| 海兰信 | 93982 | 122071 | 净卖出-28089 ⚠️ | -0.85%弱势 |
| 翰宇药业 | 224767 | 241361 | 净卖出-16594 ⚠️ | +4.07%但冲高回落 |

**注意**：外盘/内盘只是当日累计值，开盘初期数据量小参考性有限。下午盘数据更可靠。大涨+外盘>内盘=强势确认；大涨+内盘>外盘=冲高回落风险。

## PE字段用途（基本面快速排雷）

在尾盘选股等时间敏感场景中，如果iFinD MCP不可用，可借腾讯PE字段做快速基本面排雷：
- PE > 0 且 ≤ 100 → 正常盈利
- PE > 100 → 微利，高估
- PE ≤ 0 → 亏损股，基本面排雷应排除
- 注意：此字段仅供参考，不替代iFinD MCP的完整财务数据

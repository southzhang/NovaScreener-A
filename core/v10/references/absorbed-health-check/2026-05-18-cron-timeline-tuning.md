# 2026-05-18 盯盘时间线调优记录

## 背景
用户要求梳理V10+实盘盯盘的时间线，确保覆盖开盘时段且数据新鲜。

## 涉及的3个盯盘任务

| 任务 | Job ID | 类型 | 调整前 | 调整后 |
|------|--------|------|--------|--------|
| V10全扫描 | 5b9f04f29dfe | 脚本 | 25,55 9-14 | 不变 |
| V10盘中盯盘 | 995192ae60fa | 脚本 | 0,15,30,45 9-14 | **10,25,40,55 10-14** |
| 实盘盯盘 | 26daefd24af7 | Agent | 10,25,40,55 9-14 | 不变 |

## 调优逻辑

### 问题1：盯盘在扫描前就跑（数据陈旧）
- V10盯盘原来的 `0,15,30,45` 在 9:00/9:15 就跑，但V10全扫描 9:25 才出第一批信号
- 9:00 那轮盯盘推的是上周五收盘的旧数据
- **修复**：V10盯盘改到 10:10 开始，此时已有 9:25 + 9:55 两轮扫描数据

### 问题2：开盘前推送无意义
- 9:00 盯盘推飞书，用户还没开盘就收消息
- **修复**：盯盘首轮不早于开盘后10分钟

### 问题3：时间线覆盖不完整
- 09:30 开盘 → 第一次盯盘如果推迟太多（如10:10），间隔太长
- 实盘盯盘用 `10,25,40,55 9-14`，但 9:10 那一轮在集合竞价后期，数据不够稳定
- 用户认可：9:10 当热身（有数据跑，没数据静默），9:40 才是第一个有用的盯盘

## 最终时间线

```
9:10  实盘盯盘（热身，数据不稳定）
9:25  V10全扫描第1轮
9:30  ⏰ 开盘
9:40  实盘盯盘（开盘10分钟，数据稳定）
9:55  实盘盯盘 + V10全扫描第2轮
10:10 V10盯盘第1轮（推飞书）+ 实盘盯盘
10:25 全部盯盘 + V10扫描
...每15分钟交替
14:55 最后一轮
```

## 编排规则总结

1. **脚本任务优先于Agent任务**：V10全扫描（脚本，快）先跑 → 盯盘（Agent，慢）后跑
2. **上游 > 下游，留 ≥5分钟**：扫描 → 观察池 → 盯盘读观察池
3. **盯盘首轮不早于开盘后10分钟**，避免9:30开盘瞬间的虚假信号
4. **开盘前盯盘用静默模式**（cron正常跑但不推消息），避免刷屏
5. **不同任务用不同调度时间点**，避免同时并发导致MCP token争抢oad

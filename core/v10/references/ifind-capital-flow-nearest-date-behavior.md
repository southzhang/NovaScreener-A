# iFinD MCP "最新"关键词返回多行数据的行为（2026-05-19发现）

## 现象

查询"XX股票的最新主力净流入额和主力净流入额占比"时，iFinD MCP可能返回**多行数据（数组）**，而非单行最新值。

这意味着 **"最新"不等于"最近有数据的行"，而是"最近3-5个交易日的数据行"**。

## 实测案例（2026-05-19，3只股票）

### 600936 北投科技

query: `"北投科技(600936) 最新主力净流入额和主力净流入额占比"`

返回3行（实际是最近3个交易日）：
| 日期 | 主力净流入额 | 主力净流入占比 | 说明 |
|------|------------|---------------|------|
| 2026-05-19 | (空值\\t) | (空值\\t) | 盘中/未生成 |
| 2026-05-18 | -19,739,307元 | -0.2225% | ✅ 有值 |
| 2026-05-15 | +38,888,976元 | +0.4255% | ✅ 有值 |

**问题**：第1行（最新日期5/19）为空值。如果按"取第一行"的简单逻辑，就会取到空。

### 300912 凯龙高科

query: `"凯龙高科(300912) 最新主力净流入额和主力净流入额占比"`

返回：
| 日期 | 主力净流入额 | 主力净流入占比 | 说明 |
|------|------------|---------------|------|
| 未知 | -15,532,483元 | -0.4995% | ✅ 有值（未标注日期，仅返回单行） |

**注意**：凯龙高科只返回了1行有值数据且未标注日期——这是"最新"的另一种表现形式：有时返回多行（含空值），有时仅返回1行。

### 300065 海兰信

query: `"海兰信(300065) 最新主力净流入额和主力净流入额占比"`

返回3行：
| 日期 | 主力净流入额 | 主力净流入占比 | 说明 |
|------|------------|---------------|------|
| 2026-05-19 | +166,970,000元 | +1.0106% | ✅ 有值（今天） |
| 2026-05-18 | -7,758,327元 | -0.0587% | ✅ 有值（昨） |
| 2026-05-15 | +581,381元 | +0.0038% | ✅ 有值（上周五） |

**海兰信最幸运**：5/19盘中已有当日数据，所以第一行就是有效值。但同样的query写法，结果三只股票的表现完全不一样。

## 对比总结

| 股票 | 行数 | 最新行状态 | "最新"理解 |
|------|------|-----------|-----------|
| 北投科技 600936 | 3行 | ❌ 空值（5/19无数据） | 返回3天区间，含空值 |
| 凯龙高科 300912 | 1行 | ✅ 有值（但未标日期） | 只返回最新有值的行 |
| 海兰信 300065 | 3行 | ✅ 有值（5/19有数据） | 返回3天区间，全部有值 |

## 铁律

### 解析时不能取第一行

```python
# ❌ 错误：假设第一行是最新有效值
data = result.get("600936", {})
main_inflow = data.get("主力净流入额")  # 可能取到空值！

# ✅ 正确：取非空值的第一行（skip空值行）
rows = result.get("600936", [])  # 可能返回列表
if isinstance(rows, list):
    for row in rows:
        if row.get("主力净流入额") is not None and str(row.get("主力净流入额")).strip() not in ('', '\t', 'None'):
            main_inflow = row.get("主力净流入额")
            break
```

### 不要假设返回值类型

同一只股票在不同日期可能返回不同的数据结构：
- 有时返回 `list[dict]`（多行多日）
- 有时返回 `dict`（一行单日）
- 有时返回 `list[dict]` 但部分行空值

**通用解析模式**：

```python
def get_latest_value(rows, field_name):
    """从iFinD返回结果中取最新有值的数据"""
    if not rows:
        return None
    # 确保是列表
    if isinstance(rows, dict):
        rows = [rows]
    # 取第一个非空值
    for row in rows:
        val = row.get(field_name)
        if val is not None and str(val).strip() not in ('', '\t', 'None'):
            return val
    return None
```

### 采集脚本需要校验数据时效

```python
# 采集时记录实际数据日期
data_date = None
rows = result.get("600936", [])
if isinstance(rows, list):
    for row in rows:
        if row.get("主力净流入额") is not None:
            data_date = row.get("日期") or "未知"
            break

# 写入缓存时标注
{
  "last_update": "2026-05-19 15:00:00",
  "data_date": data_date,  # 标注实际数据日期
  "note": f"北投科技最新有效数据日期:{data_date}"
}
```

## 根本原因推测

iFinD "最新"关键词的行为不是"返回最新有值的数据点"，而是"返回最近3-5个交易日的数据区间（含空值）"。这与行情API的"latest"返回单行最新不同，是iFinD资金面API独有的行为。

因此：
- **不加"最新"关键词**时，iFinD可能返回更少行（更接近最新有效值）——这是之前的经验
- **加"最新"关键词**时，iFinD明确返回多日区间——但区间内含空值需要过滤
- **最佳实践**：短query（股票+指标名，无时间修饰词）+ 代码级空值过滤

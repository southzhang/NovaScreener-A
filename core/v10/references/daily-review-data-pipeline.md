# 日终复盘数据流水线

## 数据源选择（2026-05-19验证）

日终复盘（收盘后）和盘中选股/盯盘是**完全不同**的用例，数据源选择不同：

| 数据需求 | 推荐方式 | 原因 |
|----------|----------|------|
| 持仓列表 | `current_holdings.py` | 唯一持仓来源，勿硬编码 |
| 实时行情（盘后） | 腾讯 `qt.gtimg.cn` → GBK解析 | 免费、快速、无限制。MCP版也行但没必要 |
| 热门板块 | **web_search + 权威财经网站** 优先 | 市场综述文章（新浪/东方财富/财联社）直接给出当日板块排名+龙头股+涨跌幅+事件驱动原因。比MCP版块涨幅排名更详细、更可读 |
| 个股新闻 | **web_search + 东方财富/雪球/巨潮** | 搜索引擎能返回包含新闻+公告+舆情的综合结果，比MCP search_notice更全面 |
| 大盘指数 | 腾讯（通过sh000001/sz399001/sz399006获取）或 web_search | 市场综述文章直接包含指数数据+成交额 |
| 资金面数据 | MCP get_stock_performance 或 capital_flow.json缓存 | 如需精确的主力净流入数据 |
| 基本面数据 | iFinD MCP get_stock_financials 或 web_search | 定期报告数据 |

## 实践经验

### hot sectors: web_search > MCP

2026-05-19实际对比：
- **web_search** "今日A股热门板块" → 直接返回新浪财经文章，列出电力（涨停潮）、半导体、机器人三大板块，含龙头股名称+涨停板数+事件驱动原因（"算电协同"政策、Token需求激增等）
- **iFinD MCP假设结果**：返回的只是板块指数涨跌幅排名，不含事件原因和龙头股、不含涨停板分析

**结论**：日终复盘的热门板块分析，web_search搜索市场综述文章效果更好。MCP的板块排名数据适用于盘中实时决策（需要即时数据时），日终分析可以依赖市场综述。

### web_search 关键词设计

有效搜索关键词：
```
2026年5月19日 A股 热门板块 电力 半导体 机器人
今日A股热门板块 {年月}
{股票名} {代码} 今日 新闻 公告 {年月}
```

建议格式：`{年月日} A股 热门板块` + 可能的热门方向关键词（从早盘热点推断）

### 资金面缺失时的处理

cache中的capital_flow.json可能只包含部分持仓（2026-05-19实况：缓存仅含旧持仓，当前持仓无数据）。

处理策略：
- 不中断报告生成（资金面数据缺失 ≠ 报告不可用）
- 在报告中标注"无当日资金面数据"
- 如果时间充裕（15:30之后），可单独调用MCP补查

## 报告模板结构

日终复盘报告的标准模板（已验证可读性好）：

1. **持仓表现** — 表格：股票、代码、今日涨跌、收盘价、持仓盈亏、评级（🟢🟡🔴）
2. **大盘环境** — 三大指数涨跌 + 市场情绪判断（强势/震荡/弱势）
3. **热点板块** — 排名+涨幅+龙头股（来源：web_search市场综述）
4. **个股分析** — 每只持仓逐一分析：走势、新闻/公告、技术面判断、明日关注
5. **板块明日机会** — 从热门板块筛选逻辑性强的标的
6. **注意事项** — 防守位、大盘节奏、赛道轮动
7. **操作建议** — 具体到每只持仓的下一步操作

## 持仓盈亏计算

```
盈亏 = (现价 - 成本) × 持仓股数
盈亏% = (现价 / 成本 - 1) × 100%
总盈亏% = (总市值 / 总投资额 - 1) × 100%
```

## 数据时效性

- 日终复盘应在 **15:00-16:00** 生成（收盘后）
- 此时腾讯行情数据为当天收盘价
- 市场综述文章在15:00-16:00密集发布
- 资金面数据（主力净流入）在收盘后才完整可用

# 新浪K线API IP封禁 + 腾讯前复权数据陷阱（2026-05-25）

## 1. 新浪K线API 456 IP封禁

**接口：** `http://money.finance.sina.com.cn/quotes_service/api/json_v2.php/CN_MarketData.getKLineData`

**状态：** 返回HTTP 456 + 反爬HTML页面（非404，非503）

**区别：**
- 404 = API已下线（永久不可用）
- **456 = IP被反爬封禁**（可能临时性，但连续数日被封则需切换数据源）

**影响：** v10_auto_trade.py的`getKLineData`调用全部失败，导致：
- 所有卖出检查（硬止损-6%/移动止盈/通道死叉/破隧道）**全部跳过**
- 持仓风险完全失去自动监控，是严重风险

**修复方案：**
1. 将v10_auto_trade.py的get_klines函数切换到腾讯K线API
2. 或增加新浪→腾讯的fallback机制
3. 卖出检查的盈亏判断必须用模拟盘API实时数据，不能依赖K线价格

## 2. 腾讯前复权(qfq) K线数据与实际交易价格不匹配

**核心问题：** 腾讯K线API的`qfq`（前复权）模式返回的价格经过除权除息调整，与模拟盘/实盘的实际成本价和现价不一致。用前复权数据计算盈亏会得出**完全错误**的结果。

**实测案例（2026-05-25）：**

| 股票 | 前复权显示盈亏 | 模拟盘实际盈亏 | 差异原因 |
|------|--------------|--------------|---------|
| 晶方科技(603005) | **-6.17%**（触发硬止损！） | **+3.21%**（正常持有） | 前复权价格远低于实际交易价 |

**差点误卖：** 如果按前复权数据执行硬止损，会在实际盈利+3.21%时卖出，造成不必要的损失。

### 铁律

1. **判断盈亏（硬止损/移动止盈）→ 必须用模拟盘API实时盈亏数据**，不能用任何K线价格×成本价计算
2. **判断趋势信号（EMA/通道/死叉）→ 用腾讯K线未复权或前复权均可**（EMA指标不受复权影响，趋势方向一致）
3. **绝对禁止：用前复权收盘价÷买入成本价计算盈亏百分比**
4. **正确做法：** 模拟盘API返回的`income/incomeRatio`字段是唯一权威盈亏数据

### 腾讯K线接口参数选择

```python
# 前复权（用于EMA/MACD等技术指标计算）
# URL: param=sh603005,day,,,5,qfq  → qfqday字段，价格经过调整
# 用途：EMA趋势判断、通道死叉/金叉、MACD信号

# 不复权（用于与实际交易价格对比，但不推荐用于指标计算）
# URL: param=sh603005,day,,,5,     → day字段，原始价格
# 用途：验证当前价格是否与实际交易价一致
```

### 手动验证卖出信号的正确流程

当v10_auto_trade.py卖出检查因数据源问题失效时，手动验证步骤：

1. **查模拟盘API实时盈亏** → 判断硬止损(-6%)和移动止盈(+10%回落3%)
2. **用腾讯K线(不复权)获取EMA数据** → 判断通道死叉(EMA7<EMA26)和破隧道(价格<EMA120/EMA200)
3. **以模拟盘API盈亏数据为权威**，不信任K线价格计算出的盈亏

### 腾讯K线数据滞后性

盘中交易时段，腾讯K线API返回的最新K线是**前一交易日**的，不包含当日实时数据。用于EMA趋势判断时，滞后一天影响有限（EMA是慢指标）；但用于精确买卖点判断时需注意。

**与v10_realtime_scan.py的区别：** 该脚本会在盘中追加虚拟今日K线（用实时价格），因此不受此滞后影响。但v10_auto_trade.py的卖出检查使用新浪K线（现已封禁），未实现腾讯K线+今日虚拟K线方案。

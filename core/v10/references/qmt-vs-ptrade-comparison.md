# QMT vs PTrade 对比（2026-05-14）

## 结论：国金QMT适合V10策略，PTrade不适合

### 核心对比

| 维度 | QMT (迅投) | PTrade (恒生) |
|------|-----------|---------------|
| 运行环境 | 本地/自有服务器 | 券商云端服务器 |
| 编程语言 | Python + VBA，支持第三方库 | Python（功能受限） |
| 外部数据 | ✅ 可接入（miniQMT） | ❌ 封闭环境，大部分券商禁外网 |
| 品种覆盖 | 股票/期货/期权/两融/港股通/可转债 | 股票/基金/债券/两融 |
| 策略保密 | 本地运行，保密性强 | 云端运行，券商可见 |
| 7×24运行 | 需本地电脑常开 | ✅ 云端自动运行 |
| 入门难度 | 较高 | 较低（有模板） |
| 风控 | 基础，需自建 | 内置高级风控 |

### PTrade不适合的原因

V10策略链路依赖外部数据：
- iFinD MCP（基本面/资金面）
- iFinD HTTP API（分钟K线/问财/财务趋势）
- 腾讯行情（全市场扫描）

PTrade部署在券商内网，禁止外部网络访问，以上全部跑不通。

### 国金证券QMT优势

- 门槛最低：入金2万即可开通
- miniQMT直接可用：开通QMT后同步可用，不需单独申请
- 第三方库自由安装：numpy、akshare等不限制
- 本地运行：策略在自己电脑上

### 银河证券QMT的坑

- 可能要求部署在券商机房，不允许本地执行
- 第三方库需白名单
- 门槛高（10万/50万/300万不等）

### QMT接入后数据源变化

| 数据 | 现在(Mac) | QMT(Windows) |
|------|----------|-------------|
| 实时行情 | 腾讯 | xtdata |
| 日K线 | 腾讯 | xtdata |
| 分钟K线 | iFinD HTTP | xtdata |
| 基本面 | iFinD MCP | 不变 |
| 问财 | iFinD HTTP | 不变 |
| 下单 | 手动 | xtquant |

### xtdata月度quota消耗对比

QMT接入后HTTP API配额从~35%降到~5%以下（只剩问财+date_sequence+data_pool）。
